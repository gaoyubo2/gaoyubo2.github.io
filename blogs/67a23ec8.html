一、Money Robbing

A robber is planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security system connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

  1. Given a list of non-negative integers representing the amount of money of each house, determine the maximum amount of money you can rob tonight without alerting the police.

  2. What if all houses are arranged in a circle?

（a） List of non-negative integers

1.自然语言和伪代码

伪代码

自然语言描述

  1. 初始化数组 dp，其中 dp[i] 表示在考虑前 i 个房屋时，能够偷窃的最大金额。

  2. 对于前三个房屋，分别处理边界情况：

    • 如果只有一个房屋，直接偷窃该房屋。
    • 如果有两个房屋，偷窃其中金额较大的房屋。
    • 如果有三个房屋，偷窃第一个房屋和第三个房屋的较大金额。

  3. 对于第四个房屋及以后的每个房屋，考虑两种情况：

    • 如果偷窃当前房屋，则最大金额为前两个房屋的最大金额加上当前房屋的金额。
    • 如果不偷窃当前房屋，则最大金额为前一个房屋的最大金额。

  4. 动态规划递推公式： \[ dp[i]=max(dp[i−1],dp[i−2]+nums[i]) \]

  5. 最终返回数组 dp 中的最后一个元素，即表示能够偷窃的最大金额。

2.最优子结构和DP方程

最优子结构：

在这个问题中，最优子结构表示通过求解子问题的最优解，可以得到原问题的最优解。对于这个问题，最优子结构的体现在对于第 i 个房屋，它的最优解依赖于前 i-1 个房屋的最优解。每个房屋的状态（偷或不偷）都会影响到后续房屋的选择，因此问题具有最优子结构。

DP方程：

设 dp[i] 表示考虑前 i 个房屋时能够偷窃的最大金额。状态转移方程为： \[ dp[i]=max(dp[i−2]+nums[i],dp[i−1]) \]

3.算法的正确性

边界情况：

当房子数量为 0 时： \[ dp[0]=0dp[0]=0 \] 当房子数量为 1 时： \[ dp[1]=nums[0] \] 正确性证明：

从前往后计算 dp[i]，最终的结果是 dp[n]，即考虑了所有房子的情况。由于每一步都是基于前面的最优子结构计算的，所以最终的结果是全局最优解。因此，算法的正确性得到了证明。

4.算法的复杂度

  • 时间复杂度：O(n)，其中 n 为房子的数量。在动态规划的过程中，需要计算每栋房子的最大金额，因此时间复杂度为 O(n)。
  • 空间复杂度：O(n)，使用了一个大小为 n 的数组来存储中间结果 dp。

（b）In a circle

1.自然语言和伪代码

伪代码

  1. 第一种情况：偷窃第一个房屋

    • 定义数组 dp1，其中 dp1[i] 表示在考虑前 i 个房屋时，能够偷窃的最大金额。

    • 初始化 \[ dp1[0] = \text{nums}[0] \\dp1[1] = \max(\text{nums}[0], \text{nums}[1]) \]

    • 通过状态转移方程计算 \[ dp1[i] = \max(\text{dp1}[i-1], \text{dp1}[i-2] + \text{nums}[i])，其中 2 \leq i \leq n-2 \]

    • 最终结果为 dp1[n-2]，表示不偷最后一个房屋的最大金额。

  2. 第二种情况：不偷窃第一个房屋

    • 定义数组 dp2，其中 dp2[i] 表示在考虑前 i 个房屋时，能够偷窃的最大金额。

    • 初始化 \[ dp2[1] = \text{nums}[1]\\dp2[2] = \max(\text{nums}[1], \text{nums}[2]) \]

    • 通过状态转移方程计算 \[ dp2[i] = \max(\text{dp2}[i-1], \text{dp2}[i-2] + \text{nums}[i])，其中 3 \leq i \leq n-1。 \]

    • 最终结果为 dp2[n-1]，表示不偷第一个房屋的最大金额。

  3. 返回两种情况的较大值

    • 返回 \[ max(\text{dp1}[n-2], \text{dp2}[n-1]) \] 作为最终结果。

2.最优子结构和DP方程

最优子结构：

在计算 dp1[i] 和 dp2[i] 时，考虑两种选择：抢劫第 i 栋房子或者不抢劫。如果抢劫，那么最优选择是前两栋房子的最大金额加上当前房子的金额，即 \[ dp1[i]=dp1[i−2]+nums[i] \] 如果不抢劫，那么最优选择是前一栋房子的最大金额，即 \[ dp1[i]=dp1[i−1] \] 同理，对于 dp2[i] 的计算也是类似的。因此，问题具有最优子结构。

DP 方程：

根据最优子结构，得到动态规划方程： $$ dp1[i]=max(dp1[i−1],dp1[i−2]+nums[i])\

dp2[i]=max(dp2[i−1],dp2[i−2]+nums[i]) $$

3.算法的正确性

边界情况：

当房子数量为 0 时： \[ dp1[0]=0dp1[0]=0，dp2[0]=0dp2[0]=0 \] 当房子数量为 1 时： \[ dp1[1]=nums[0]dp1[1]=nums[0]，dp2[1]=nums[1]dp2[1]=nums[1] \] 正确性证明：

对于 dp1，计算到最后一栋房子时，即计算了前 n−2 栋房子的最大金额，由于最后一栋房子不能抢，所以结果为 ： \[ dp1[*n*−2] \] 对于 dp2，计算到最后一栋房子时，即计算了前n−1 栋房子的最大金额，由于最后一栋房子不能抢，所以结果为： \[ dp2[*n*−1] \] 最终的答案是这两种情况的最大值 \[ max(dp1[n−2],dp2[n−1]) \] 因此，算法的正确性得到了证明。

4.算法的复杂度

  • 时间复杂度：O(n)，其中 n 为房子的数量。在动态规划的过程中，需要计算每栋房子的最大金额，因此时间复杂度为 O(n)。
  • 空间复杂度：O(n)，使用了两个大小为 n 的数组来存储中间结果 dp1 和 dp2。

二、Ugly Number

An ugly number is a positive integer whose prime factors are limited to 2, 3, and 5. Given an integer n, return the nth ugly number.

  1. Using a brute-force algorithm to solve this problem, analyze the time complexity of your implemented brute-force algorithm and explain why the algorithm’s time complexity is O(n^2), where n is the number of points.

  2. Propose an improved algorithm to solve this problem with a time complexity better than thebrute-force algorithm. Describe the algorithm’s idea and analyze its time complexity.

（a）Brute-force algorithm

自然语言和伪代码

伪代码

自然语言描述

是从 1 开始逐个判断每个数字是否是丑数，直到找到第 n 个丑数为止。

定义一个辅助函数 IsUgly(num)，该函数用于判断一个数是否是丑数，即其质因数只包含 2、3 和 5。在主函数中，从 1 开始逐个判断数字是否是丑数，直到找到第 n 个丑数为止。

时间复杂度

对于每个数字 i，需要调用 IsUgly 函数进行判断，而 IsUgly 函数的执行时间最坏情况下为 O(i)。

因此，总的时间复杂度为 O(1 + 2 + 3 + ... + n) = O(n^2)。

（b）improved algorithm

1.自然语言和伪代码

伪代码

自然语言描述

维护三个指针 index2、index3、index5 和对应的因子 factor2、factor3、factor5，分别表示当前已知的丑数乘以 2、3、5 后的结果。使用动态规划的思想，从已有的丑数中生成新的丑数，直到生成第 n 个丑数为止。

具体步骤如下：

  1. 初始化数组 ugly，用于存储生成的丑数，以及三个指针 index2、index3、index5 分别指向 ugly 中的位置，和三个因子 factor2、factor3、factor5 分别表示当前已知的丑数乘以 2、3、5 后的结果。
  2. 从第二个丑数开始，使用三个指针指向的因子生成新的丑数，并更新指针和因子。
  3. 重复步骤 2 直到生成第 n 个丑数。

2.最优子结构和DP方程

对于本问题，每个丑数都是前面的丑数乘以2、3或5得到的。因此，问题的最优解可以由之前的最优解推导而来。

动态规划方程：

设 ugly[i] 表示第 i 个丑数，其值为 \[ min(factor2, factor3, factor5) \] 其中，factor2、factor3、factor5 分别表示乘以2、3、5得到的下一个丑数。

初始条件：ugly[0] = 1，表示第一个丑数为1。

状态转移方程： \[ ugly[i] = min(factor2, factor3, factor5) \] 其中，factor2、factor3、factor5 的更新规则如下： \[ factor2 = 2 * ugly[index2]，其中 index2 表示上一个乘以2得到的丑数的位置\\ factor3 = 3 * ugly[index3]，其中 index3 表示上一个乘以3得到的丑数的位置\\ factor5 = 5 * ugly[index5]，其中 index5 表示上一个乘以5得到的丑数的位置\\ \] 更新 index2、index3、index5 的规则是选择 factor2、factor3、factor5 中最小的一个，然后将对应的索引加1。

3.算法的正确性

考虑两个关键步骤：初始条件和状态转移方程。

初始条件：

对于第一个丑数，ugly[0] 被初始化为1，这是合理的，因为1是最小的丑数。

状态转移方程：

  • 对于每个 ugly[i]，其值是通过比较 factor2、factor3、factor5 的最小值得到的。这确保了每个丑数都是由之前的丑数乘以2、3或5得到的最小值。
  • 更新 factor2、factor3、factor5 的规则是选择对应的因子中最小的一个，并将对应的索引加1。这保证了每个因子都被充分考虑，并且每个丑数都是由之前的丑数生成的最小值。

综合：

通过正确的初始条件和状态转移方程，可以确保每个丑数都是通过之前的丑数生成的最小值，从而保证了整个算法的正确性。

数学归纳法：

可以通过数学归纳法来进一步证明。假设 ugly[0], ugly[1], ..., ugly[k] 都是正确的丑数序列，那么需要证明 ugly[k+1] 也是丑数。这是通过比较 factor2、factor3、factor5 的最小值得到的，而这三个值都是之前的丑数乘以2、3或5得到的最小值。因此，ugly[k+1] 也是正确的丑数。

综上所述，该动态规划算法是正确的。

4.算法的复杂度

时间复杂度：

  • 在主循环中，执行了 n 次迭代，每次迭代都涉及到常数时间的操作。
  • 因此，总体时间复杂度是 O(n)。

空间复杂度：

  • 使用了额外的空间来存储丑数序列 ugly[]，大小为 n。
  • 因此，空间复杂度是 O(n)。

三、Unique Binary Search Trees

Given n, how many structurally unique BST’s (binary search trees) that store values 1...n?

Note: Given n = 3, there are a total of 5 unique BST’s:

1.自然语言和伪代码

伪代码

自然语言描述

  1. 创建一个数组 dp，其中 dp[i] 表示包含 i 个节点的 BST 的数量。

  2. 初始化 dp[0] = 1，因为空树也是一种 BST。

  3. 从 1 开始迭代到 n，对于每个 i，计算 \[ dp[i] \] 对于每个 j，其中 j 从 1 到 i： \[ dp[i] += dp[j-1] * dp[i-j] \] 这是因为当节点值为 j 时，左子树包含了 j-1 个节点，右子树包含了 i-j 个节点，总的 BST 数量为两者的乘积。

  4. 最终，dp[n] 即为所求的结果。

2.最优子结构和DP方程

最优子结构：

问题的最优解可以通过子问题的最优解来构造，每个子问题都是原问题规模的一个部分，且子问题之间相互独立。

DP方程：

设dp[i]表示存储值1到i的BST的数量，那么对于每个i，可以将其作为根节点，左子树由1到i-1构成，右子树由i+1到n构成。因此，dp[i]可以通过累加所有可能的左右子树组合得到。对于每个j： \[ dp[i] += dp[j] * dp[i-j-1] \] 其中，dp[j]表示左子树的BST数量，dp[i-j-1]表示右子树的BST数量。因为BST的结构，左子树的BST数量乘以右子树的BST数量就是以j为根节点时的BST数量。对所有可能的j求和即可得到以i为根节点时的BST数量。

这样，可以通过迭代填充数组dp得到问题的最终解。

3.算法的正确性

  1. 基础步骤： 当 n = 0 时，空树是唯一的一棵二叉搜索树，因此 dp[0] = 1。
  2. 归纳假设： 假设对于任意的 i， dp[i] 存储了存储值 1...i 的唯一二叉搜索树的数量。
  3. 归纳步骤： 要证明 dp[i+1] 也满足条件，考虑以 i+1 为根的情况。对于 i+1，可以将其作为根，然后左子树可以是 1, 2, ..., i，右子树可以是 i+2, i+3, ..., n。由于左子树和右子树的形状是独立的，可以用 dp[i] 表示左子树的数量，dp[n-i-1] 表示右子树的数量。因此，以 i+1 为根的唯一二叉搜索树数量是 dp[i] * dp[n-i-1]。

通过数学归纳法的步骤，证明了对于任意 n，dp[n] 存储了存储值 1...n 的唯一二叉搜索树的数量。

4.算法的复杂度

  • 时间复杂度： 在算法中，使用一个循环从 2 到 n 来计算 dp 数组。在每次迭代中，需要计算 dp[i]，其中 i 的范围是 2 到 n。对于每个 i，需要计算 dp[i] 的值，这涉及到两个嵌套的循环，其中最内层的循环最多运行 i 次。因此，总体时间复杂度为 O(n^2)。
  • 空间复杂度： 使用一个长度为 n+1 的数组 dp 来存储中间结果，因此空间复杂度为 O(n+1)，即 O(n)。

四、Largest Divisible Subset

Given a set of distinct positive integers, find the largest subset such that every pair (Si, Sj ) of elements in this subset satisfies: Si%Sj = 0 or Sj%Si = 0. Please return the largest size of the subset.

Note: Si%Sj = 0 means that Si is divisible by Sj .

1.自然语言和伪代码

伪代码

自然语言描述

  1. 对输入数组 nums 进行排序，以确保之后的动态规划过程中能够满足题目条件。
  2. 创建两个数组 dp 和 parent，分别用于记录以每个元素结尾的最大可整除子集的大小和其上一个元素的索引。
  3. 使用两层循环遍历数组，对于每对元素 (nums[i], nums[j])，如果 nums[i] 能够整除 nums[j] 且以 nums[j] 结尾的子集大小加一大于以 nums[i] 结尾的子集大小，更新 dp[i] 和 parent[i]。
  4. 在更新过程中，记录最大子集的大小和结束索引。
  5. 通过最大子集的结束索引反向遍历 parent 数组，构建最大可整除子集。
  6. 返回最大可整除子集。

2.最优子结构和DP方程

设 dp[i] 表示以 nums[i] 结尾的最大整除子集的长度，parent[i] 表示 nums[i] 在最大整除子集中的前一个元素的下标。

则有最优子结构关系： \[ dp[i]=max(dp[j]+1), where 0≤j<i \and nums[i]\mod nums[j]=0 \] 其中，初始化时 dp[i] = 1，表示每个元素自身构成一个大小为 1 的整除子集。

此外，需要维护 maxSize 和 maxIndex 来记录全局最大的整除子集长度和对应的结束元素下标。

最后，从 maxIndex 开始回溯得到最大整除子集。

3.算法的正确性

1. 基本情况的正确性：

当问题规模为 1 时，只有一个元素，显然符合条件，所以算法的基本情况是正确的。

2. 归纳步骤的正确性：

假设算法对于规模为 k 的问题是正确的，即对于任意 1 到 k 的元素集合，算法都能正确地找到最大的可整除子集。

现在考虑规模为 k+1 的问题。对于元素集合 {a1, a2, ..., ak, ak+1}，算法对于 {a1, a2, ..., ak} 的子集是正确的。现在要添加 ak+1。

使用动态规划数组 dp 来跟踪以每个元素结尾的最大可整除子集的大小。在遍历到 ak+1 时，检查 ak+1 能够整除之前的元素中的哪些元素。选择将 ak+1 加入到能够得到最大 dp[i] 的那个子集中。由于假设对于规模为 k 的问题是正确的，所以这一步的操作是正确的。

综上所述，通过数学归纳法，证明了算法对于所有规模的问题都是正确的。

4.算法的复杂度

时间复杂度：

  • 排序： 算法开始时对输入数组进行排序，排序的时间复杂度是 O(nlogn)。
  • 动态规划： 在动态规划的过程中，需要遍历每个元素，对于每个元素，都需要检查其是否能整除之前的元素，这个过程的时间复杂度是 O(n^2)。
  • 综合考虑排序和动态规划，算法的总体时间复杂度是 O(nlogn) + O(n^2) = O(n^2)。

空间复杂度：主要由动态规划数组 dp 和结果数组 result 决定，都是 O(n)

五、Target Sum

You are given an integer array nums and an integer target. You want to build an expression out of nums by adding one of the symbols ’+’ and ’-’ before each integer in nums and then concatenate all the integers. For example, if nums = [2, 1], you can add a ’+’ before 2 and a ’-’ before 1 and concatenate them to build the expression ”+2-1”.Return the number of different expressions that you can build, which evaluates to target.

Example: Input: nums = [1,1,1,1,1], target = 3

Output: 5 Explanation: There are 5 ways to assign symbols to make the sum of nums be target 3. -1 + 1 + 1 + 1 + 1 = 3 +1 - 1 + 1 + 1 + 1 = 3 +1 + 1 - 1 + 1 + 1 = 3 +1 + 1 + 1 - 1 + 1 = 3 +1 + 1 + 1 + 1 - 1 = 3

1.自然语言和伪代码

伪代码

自然语言描述

  1. 首先，计算数组 nums 的总和，并判断是否存在整数 targetSum，使得 (sum + target) / 2 = targetSum。
  2. 如果不存在，则直接返回 0，因为无法找到有效的表达式。
  3. 使用动态规划数组 dp，其中 dp[j] 表示累积和为 j 的表达式的数量。
  4. 初始化 dp[0] 为 1，表示累积和为 0 的表达式有 1 种方式。
  5. 遍历数组 nums，更新动态规划数组。对于每个数字 num，从 targetSum 向 num 遍历，更新 dp[j]。
  6. 最终，dp[targetSum] 即为满足条件的表达式数量，返回该值。

2.最优子结构和DP方程

在这个问题中，定义状态 dp[j] 表示在考虑数组 nums 时，能够得到和为 j 的不同表达式的数量。

其中，第 i 个元素有两种选择，分别是添加正号或负号。

DP方程： \[ dp[j]=dp[j−nums[i]]+dp[j] \] 初始化条件为： \[ dp[0]=1 \] 对于 i = 0 时，表示在考虑空数组时，和为 j 的表达式数量为 0。

整体的动态规划数组 dp 的大小为 \[ (\text{targetSum} + 1) \] 其中 targetSum 是目标和的一半。

3.算法的正确性

  1. 基础步骤： 当数组为空时，目标值为0，算法返回1。这是正确的，因为只有一种表达式：空表达式。因此，基础步骤成立。

  2. 归纳假设： 假设算法在数组大小小于等于 k 时得到正确的结果，其中 k 是数组的大小。

  3. 归纳步骤： 考虑数组大小为 k+1 时的情况。遍历数组中的每个数字，并计算目标值的表达式数量。对于数组中的第 k+1 个数字 num，考虑两种情况：包含 num 和不包含 num。

    • 包含 num： 更新动态规划数组 dp[j]，其中 j 表示目标和。将 dp[j] 更新为 \[ dp[j] + dp[j-\text{num}] \] 表示在包含 num 的情况下得到和为 j 的表达式数量。

    • 不包含 num： 此时 dp[j] 保持不变。

    通过上述两种情况的组合更新了整个动态规划数组。根据归纳假设，数组大小为 k 时的结果是正确的。因此，通过递推关系得到了数组大小为 k+1 时的正确结果。

根据数学归纳法，得知算法对于任何大小的数组都能够给出正确的结果。

4.算法的复杂度

  • 时间复杂度： O(n * target)

    其中 n 是数组的长度，target 是目标值。

  • 空间复杂度： O(target)

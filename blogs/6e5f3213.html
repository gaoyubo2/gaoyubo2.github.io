puzzle2：共享变量识别

通过插桩，使得程序能够在运行时识别线程间共享的全局变量（被多于一个线程读或写的全局变量），并在程序退出前输出这些线程间共享的全局变量的符号名。 输出格式应为："shared variables: var0, var1, var2"。 注意：为降低难度，本题中全局变量仅包含基本类型（int、double、bool、pointer等），不包含数组等复杂结构。

提示1：插桩读取指令（LoadInst）和存储指令（StoreInst），识别程序读/写操作对应的内存地址。通过判断同一个内存地址是否在两个不同的线程中被读/写，来判断该内存地址对应的变量是否是线程间共享的变量。

提示2：可以借助pthread_self()函数，通过观察其返回值确定当前线程。不同线程pthread_self()的返回值一定不同，相同线程pthread_self()的返回值一定相同。

提示3：对于一个LLVM变量（llvm::Value），在插桩时可以通过llvm::Value::getName方法获取其符号名。静态变量以及栈上变量一定能获取到符号名，堆上变量不一定有符号名（即获取到的符号名为""）。

提示4：在插桩时，可以使用IRBuilder<>::CreateGlobalStringPtr方法向程序中插入一个静态字符串。此方法接受一个字符串，返回一个指向llvm::Value*类型变量的指针。该llvm::Value变量可视为指向静态字符串的起始地址的指针在LLVM IR系统中的占位符。使用该llvm::Value变量作为函数参数，创建函数调用，将新插入的静态字符串的起始地址传递给新插入的插桩函数，从而在程序运行时获取变量的符号名。

插桩思路

查找插桩点的方法

  1. 遍历函数的基本块和指令：
    • 在MyPass.cpp的run函数中，通过for循环遍历函数F的每个基本块。
    • 在每个基本块内部，再通过嵌套的for循环遍历基本块内的指令。
  2. 判断是否是插桩点：
    • 对于每个指令，首先使用dyn_cast函数尝试将其转换为LoadInst类型的指令。
    • 如果转换成功，表示当前指令是LoadInst类型的指令。
    • 接下来，使用getPointerOperand方法获取该LoadInst指令的操作数，即被加载的指针。
    • 使用dyn_cast函数再次尝试将操作数转换为GlobalVariable类型的全局变量。
    • 如果转换成功，表示该指令是插桩点，因为它加载了一个全局变量。

在插桩点进行插桩的手法

  1. 创建插桩函数：
    • 在MyPass.cpp中，首先需要创建一个插桩函数，用于记录共享变量的访问情况。
    • 使用getOrInsertFunction函数获取或创建插桩函数的函数指针。这个函数的名称是"record_access"，接受一个uint64_t类型的线程ID和一个const char*类型的变量名作为参数，并返回void类型。
    • 如果该插桩函数已经存在，直接获取其函数指针；如果不存在，则创建一个新的函数，并设置其函数类型和参数类型。
  2. 在插桩点之前插入代码：
    • 在确定了插桩点后，使用IRBuilder对象在插桩点之前插入代码。
    • 首先，创建一个IRBuilder对象，使用它来构建新的指令。
    • 调用SetInsertPoint方法，将插桩点设置为当前指令，以确保新的指令插入到正确的位置。
  3. 创建插桩函数的参数：
    • 在插桩函数中，需要传递两个参数：当前线程的ID和共享变量的名称。
    • 使用CreateCall函数调用pthread_self函数获取当前线程的ID，将其作为插桩函数的第一个参数。
  4. 创建共享变量名称的指针：
    • 使用CreateGlobalStringPtr函数创建全局变量名称的指针，将其作为插桩函数的第二个参数。
    • CreateGlobalStringPtr函数将字符串转换为GlobalVariable类型的指针，用于在插桩函数中获取共享变量的名称。
  5. 插入插桩函数的调用指令：
    • 使用CreateCall函数创建插桩函数的调用指令。将插桩函数的函数指针和参数传递给CreateCall函数。
    • 这样，插桩函数就会在插桩点处被调用，并记录共享变量的访问情况。

数据记录和输出的方法

  1. 数据记录：
    • 在MyFunc.cpp中，定义了一个全局变量records，它是一个std::map。
    • records用于记录共享变量的访问情况。其中，键是变量名，值是一个std::set。
    • std::set用于存储访问该变量的线程ID，确保每个线程ID只记录一次。
    • 在插桩函数record_access中，记录访问共享变量的线程ID。
  2. 插桩函数record_access：
    • record_access函数是在插桩点处调用的函数，用于记录共享变量的访问情况。
    • record_access函数接受两个参数：线程ID和共享变量的名称。
    • 在函数内部，首先在records中查找变量名称对应的记录。
    • 如果找到了记录，将线程ID插入到对应的std::set中。
    • 如果没有找到记录，则创建一个新的std::set，并将线程ID插入其中。
    • 这样，就实现了在插桩点记录共享变量访问的功能。
  3. 输出访问了多个线程的共享变量：
    • 在MyFunc.cpp中，定义了一个print_shared_variables函数，用于打印访问了多个线程的共享变量。
    • print_shared_variables函数遍历records，对于记录中访问次数大于1的变量，将其打印出来。
    • 首先，遍历records中的每个键值对，获取变量名称和对应的std::set。
    • 对于每个键值对，判断std::set的大小是否大于1，即是否有多个线程访问了该变量。
    • 如果是，将变量名称打印出来，表示访问了多个线程的共享变量。

puzzle3：lockset算法

通过插桩，使得程序能够在运行时识别存在数据竞争的全局变量（没有被同一个互斥锁保护、被多于一个线程读或写、且至少被一个线程写的全局变量），并在程序退出前输出这些存在数据竞争的全局变量的符号名。 输出格式应为："race variables: var0, var1, var2"。

插桩思路

查找插桩点的方法

  1. 遍历函数的基本块和指令：
    • 在 MyPass.cpp 的 run 函数中，通过嵌套的 for 循环遍历函数 F 的每个基本块。
    • 在每个基本块内部，再通过嵌套的 for 循环遍历基本块内的指令。
  2. 判断是否是插桩点：
    • 对于每个指令，首先使用 dyn_cast 函数尝试将其转换为 LoadInst、StoreInst 或 CallInst 类型的指令。
    • 对于 LoadInst 和 StoreInst，再次使用 dyn_cast 函数尝试将其操作数转换为 GlobalVariable 类型的全局变量。如果转换成功，表示该指令是插桩点，因为它加载或存储了一个全局变量。
    • 对于 CallInst，直接获取被调用的函数，并判断其是否是 pthread_mutex_lock 或 pthread_mutex_unlock 函数。如果是这些函数之一，该调用指令也被视为插桩点。
  3. 插入插桩操作：
    • 一旦确定了插桩点，使用 IRBuilder 创建相应的插桩函数的参数。
    • 使用 CreateCall 插入对插桩函数的调用，以在这些位置插入相应的插桩操作。

在插桩点进行插桩的手法

  1. 插桩函数的创建和参数：
    • record_access：记录访问操作的相关信息。
      • 参数：thread_id（线程ID）、var_name（变量名）、is_write（是否写操作）、is_locked（是否已加锁）。
    • print_race_variables：打印存在竞争条件的变量。
    • record_lock：记录互斥锁的加锁操作。
      • 参数：thread_id（线程ID）、mutex_id（互斥锁ID）。
    • record_unlock：记录互斥锁的解锁操作。
      • 参数：thread_id（线程ID）、mutex_id（互斥锁ID）。
  2. 插桩点的插入：
    • 在加载操作（LoadInst）和存储操作（StoreInst）之前插入代码，以记录访问操作。
    • 在调用 pthread_mutex_lock 和 pthread_mutex_unlock 函数之前插入代码，以记录互斥锁的操作。
  3. print_race_variables 函数的调用：
    • 在 main 函数结束时插入对 print_race_variables 函数的调用，以打印存在竞争条件的变量。
  4. 全局数据结构和互斥锁：
    • 定义了一个全局变量 records，用于记录访问和锁的状态。
    • 定义了一个全局变量 locksets，用于记录线程的锁集。
    • 使用互斥锁 race_mutex 来保护全局数据结构的访问。
  5. 竞争条件的检测：
    • 在 print_race_variables 函数中，遍历 records 中的访问记录，检查是否存在竞争条件的变量。
    • 如果一个变量有多个线程进行写操作，并且这些线程没有共享的锁，那么就认为存在竞争条件。

数据记录和输出的方法

  1. 数据记录：
    • 在代码中定义了一个全局变量records，它是一个std::map，用于记录共享变量的访问情况。
    • records的键是变量名，值是一个std::set<uint64_t>，用于存储访问该变量的线程ID。
    • 在函数record_access中，记录访问共享变量的线程ID。
    • record_access函数接受两个参数：线程ID和共享变量的名称。
    • 在函数内部，首先在records中查找变量名称对应的记录。
    • 如果找到了记录，将线程ID插入到对应的std::set<uint64_t>中。
    • 如果没有找到记录，则创建一个新的std::set<uint64_t>，并将线程ID插入其中。
    • 这样，就实现了在插桩点记录共享变量访问的功能。
  2. 输出访问了多个线程的共享变量：
    • 在函数print_shared_variables中，遍历records，对于记录中访问次数大于1的变量，将其打印出来。
    • 首先，将records按照键的字典序进行排序，得到一个新的std::map对象sorted_records。
    • 遍历sorted_records中的每个键值对，获取变量名称和对应的std::set<uint64_t>。
    • 对于每个键值对，判断std::set<uint64_t>的大小是否大于1，即是否有多个线程访问了该变量。
    • 如果是，将变量名称打印出来，表示访问了多个线程的共享变量。

puzzle1

给定一个包含n+1个整数的数组nums，其数字都在[1,n]之间（包括1和n），可知至少存在一个重复的整数

思路

可以将数组看作是一个链表，数组中的每个元素都指向下一个元素的索引位置。

使用两个指针，一个指针移动速度较快（每次移动两步），另一个指针移动速度较慢（每次移动一步）。如果存在循环，那么快指针最终会追上慢指针，并且二者会在某个位置相遇。

一旦快慢指针相遇，就可以确定存在循环。接下来，再次将其中一个指针（比如慢指针）指向数组的起始位置，并让两个指针以相同的速度移动，每次移动一步。当它们再次相遇时，就意味着它们已经到达循环的起点。

  1. 初始化指针：
    • 创建两个指针slow和fast，初始值都为nums[0]。
    • 这两个指针将在后续的循环中以不同的速度移动。
  2. 寻找循环起点：
    • 使用do-while循环，不断移动指针直到它们相遇。
    • 在循环中，slow指针每次移动一步，即slow = nums[slow]。
    • fast指针每次移动两步，即fast = nums[nums[fast]]。
    • 当两个指针相遇时，表示存在一个循环，并且在循环中的某个位置。
  3. 确定循环起点：
    • 将一个指针ptr1重新指向数组的起始位置，即nums[0]。
    • 将另一个指针ptr2指向之前相遇的位置，即slow。
    • 然后，将两个指针以相同的速度移动，每次移动一步，直到它们再次相遇。
    • 当两个指针再次相遇时，表示它们已经到达循环的起点。
  4. 返回重复的数字：
    • 由于循环的起点即为重复的数字，所以返回ptr1或ptr2都可以。
    • 返回其中任意一个指针所指向的元素即可。

puzzle2

给定一个全是正整数的数组nums，现在假设nums中只有一个数字出现了奇数次，请返回这个整数的值

思路

  1. 初始化结果变量：
    • 创建一个整数变量result，并将其初始值设为0。
    • result将用于存储出现奇数次的数字。
  2. 遍历数组：
    • 使用for-each循环遍历给定的整数数组nums。
    • 在每次循环中，将当前数字num与result进行异或操作。
  3. 异或操作的作用：
    • 异或操作（^）是一种位运算，当两个操作数的对应位不同时，结果为1，否则结果为0。
    • 初始时，result为0，所以与任何数异或的结果都是那个数本身。
    • 当出现重复的数字时，异或操作会将重复的数字抵消，结果为0。
    • 对于出现奇数次的数字，由于其他数字都是成对出现的，所以异或操作不会将其抵消，结果将是这个奇数次出现的数字。
  4. 返回结果：
    • 在完成整个数组的遍历后，result中存储的将是出现奇数次的数字。
    • 将result作为函数的返回值，即找到了数组中出现奇数次的数字。

puzzle3

Bonus：C老师最近沉迷比特币，众所周知虚拟货币的价格波动性很大，所以在实际买入之前，C老师想通过模拟交易寻找能获利的交易策略。

现在C老师收集了过去一段时间内的比特币价格变动数据prices（即每天的最高价），为简化交易策略C老师选择梭哈（即每次买入交易之前会卖出所持有的所有比特币来回笼资金，且每次交易都只买卖一个比特币），现在请设计一个函数用来计算C老师模拟交易能获得的最大利润

思路

  1. 初始化最大利润变量为0。
  2. 遍历价格变动数据的每一天（从第二天开始）：
    • 比较当前天的价格与前一天的价格。
    • 如果当前天的价格高于前一天的价格，表示可以进行交易并获利。
    • 计算当前交易的利润（即当前天的价格减去前一天的价格）。
    • 将当前交易的利润累加到最大利润变量中。
  3. 返回最大利润。

这个思路基于以下观察：

  • 在梭哈交易策略下，为了获取最大利润，可以在每次价格上升时进行买卖操作。
  • 当价格连续上升时，每次买卖操作都可以获得利润。
  • 所以，只需要关注价格上升的阶段，并将每个阶段的利润累加起来，即可得到总体的最大利润。

这个问题可以使用动态规划来解决。

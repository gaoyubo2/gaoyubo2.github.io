搜索引擎的原理

  1. 蜘蛛在互联网上爬行和抓取网页信息，并存入原始网页数据库
  2. 对收集回来的网页进行分析，提取相关网页信息
  3. 根据一定的相关度算法进行大量复杂计算，得到每一个网页针对页面内容中及超链中每一个关键词的相关度（或重要性）
  4. 用这些相关信息建立网页索引数据库
  5. 根据用户输入的关键词，快速找到相关文档，并对找到的结果进行排序，并将查询结果返回给用户

搜索引擎的发展阶段

第一代搜索引擎出现于1994年，主要特征为集中式检索

第二代搜索引擎大约出现在1996年，大多采用分布式检索方案。

第三代搜索引擎：个性化、专业化智能化，规模继续增大

第四代搜索引擎：从信息检索到信息推送的转变

什么是网络爬虫

网络爬虫是一种通过既定规则（网络爬虫算法），能够自动提取网页信息的程序,是搜索引擎的重要组成部分。

通用网络网络爬虫的基本工作流程

  • 获取初始的URL，该URL地址是用户自己制定的初始爬取的网页。
  • 爬取对应URL地址的网页时，获取新的URL 地址。
  • 将新的URL地址放入URL队列。
  • 从URL队列中读取新的URL，然后依据新的 URL爬取网页，同时从新的网页中获取新的URL地址， 重复上述的爬取过程。
  • 设置停止条件

传统爬虫的基本流程

  • 发起请求
  • 获取响应内容
  • 解析内容
  • 保存数据

HTTP和HTTPS协议

HTTP：超文本传输协议，是一种一次性连接，端口号80

HTTPS：在HTTP协议基础上添加了安全套接字协议 （Secure Sockets Layer，SSL），端口号为443

Web服务器的工作原理

  1. 建立连接：客户端通过TCP/IP协议建立到服务器的TCP连接。
  2. 请求过程：客户端向服务器发送HTTP协议请求包（ Request ）， 请求服务器里的资源文档。
  3. 应答过程：服务器向客户端发送HTTP协议应答包（ Response ）， 如果请求的资源包含有动态语言的内容， 那么服务器会调用动态语言的 解释引擎负责处理“动态内容”，并将处理后得到的数据返回给客户端。 由客户端解释HTML 文档，在客户端屏幕上渲染图形结果。
  4. 关闭连接：客户端与服务器断开。

Request请求方式

  • GET是从服务器上获取信息，而且是安全的和幂等的
  • POST是向服务器传送数据，可用于更新站点
区别
  • GET请求：请求的数据会附加在URL之后，以?分割URL和传输数据，多个参数用&连接。URL的编码格式采用的是ASCII编码，数据会暴露(明文)在地址栏中
  • POST请求：POST请求会把请求的数据放置在HTTP请求包的包体中，需要构造一个表单，通过表单提交得到POST请求。

分词

​ 将一个中文文本分割成若干个独立的、有意义的基本单位的过程

中文分词的难点

  • 分词歧义消解
  • 未登录词的识别
  • 错别字、谐音字规范化
  • 分词粒度问题

分词方法

逆向匹配的切分精度略高于正向匹配

  • 一般分词方法
    • 基于词典的分词：词典中的词条
    • 基于理解的分词：通过语义信息
    • 基于统计的分词：出现的频率
  • 正向最大匹配分词（FMM）
    • 优先匹配最长词汇
    • 失败则从段末尾减去一个字，再匹配
    • 单纯使用错误率：1/169
  • 逆向最大匹配法（BMM）
    • 从句子末尾开始
    • 失败则去掉前面的一个汉字
    • 单纯使用错误率：1/245

算法

① 首先用正则表达式将中文段落粗略的分成一个个句子。

② 将每个句子构造成有向无环图，之后寻找最佳切分方案。

③ 最后对于连续的单字，采用HMM模型将其再次划分。

统计分词

利用字与字相邻出现的频率来反映成词的可靠度

步骤
  1. 建立统计语言模型
  2. 对句子进行单词划分，然后对划分结果进行概率计算

主要思想：词是稳定的组合,不需要切分词典，故可以实现无词典分词

  • 隐 Markov 模型

    • 解码问题

  • N-gram模型

TF

信源编码

  1. 抽样：抽样频率应选为f s≥2fm。
  2. 量化：抽样得到的瞬时值将其幅度离散
    • 均匀量化
    • 非均匀量化

  1. 设法减少码元数目和降低码元速率，即通常所说的数据压缩；
  2. 将信源的模拟信号转化成数字信号，以实现模拟信号的数字化传输。

  • 参数编码：是以人的发音机制的模型作为基础，对语音信号特征参数的提取后进行编码，质量中，效率高
    • 线性预测编码LPC

无线通信信道

为了保证通信的可靠性，必须采取信道编码：以最小的监督码元，获得最大检错和纠错能力。

  • 通过编码加入冗余度（称为监督码元）
  • 可以检测和纠正接收信息流中的差错，提高可靠性

码距

两个码组中对应码元位置上取值不同的位数，称为码距

  • 为了检测e个错码，需要最小码距：d >= e+1
  • 为了纠错t个错码，需要最小码距：d >= 2t+1
  • 为了检测e个错码，纠正t个编码，需要最小码距：d >= e+t+1

主要的信道编码方式

  • 奇偶校验码
  • 线性分组码：信息码元与监督码元之间具有线性关系
    • 卷积码：是一种非分组码，一般表示为(n,k,m)
    • Turbo码：又称并行级联卷积码(PCCC)，将卷积码和随机交织器结合在一起， 实现了随机编码的思想。为了提高码率,需要经过删余器， 采用删余
  • 熵编码
    • 香农(Shannon)编码
    • 哈夫曼(Huffman)编码
    • 算术编码 (arithmetic coding)

调制技术

调制是指用待传输的信号控制另外一个便于传输的载波信号的某一个参数的变化，以便达到传输信号的目的。

  • 调制信号：待传输的信号
  • 已调信号：调制之后的信号
  • 解调：调制的逆过程

调制方式的划分

  • 模拟调制
    • 幅度调制 （AM）
    • 频率调制（FM）
    • 相位调制（PM）
  • 数字调制：根据其改变载波方式的不同：
    • 幅度键控（ASK）
    • 频移键控（FSK）
    • 相移键控（PSK）
    • 正交幅度调制法（QAM）：载波相位发生变化，而且幅度也变化。

各种调制之间的差别都反映在 同相分量I(t) 和 正交分量Q(t) 上

二进制调制

二进制振幅键控（2ASK)

利用载波的幅度变化来传递数字信息的，其频率和初始相位保持不变

二进制频移键控（2FSK）

利用载波的频率变化来传递数字信息的

二进制相移键控（2PSK）

利用载波的相位变化来传递数字信息，而振幅和频率保持不变。

多进制数字调制

多址技术

FDMA-频分多址

在频分双工（FDD：frequency division duplex）系统中，分配给用户一个信道，即一对频道

TDMA-时分多址

一个信道由一连串周期性的时隙所构成，允许多个用户在不同的时隙来使用相同的频率

帧结构：由时隙组成，在时隙内传送的信号叫做突发（burst）

CDMA-码分多址

（Code Division Multiple Access）

每一个用户信号被分配一个自相关性大而互相关性小的伪随机二进制序列进行扩频，这个伪随机二进制序列就被称为地址码（即PN码）；

地址码之间具有相互准正交性

许多用户的宽带信号相互叠加在一起同时进行宽带传输。

CDMA码序列

对于扩频码与地址码的要求：良好的伪随机性能和相关性能

  • 选择互相关性好的编码作为地址码：最常用的就是Walsh码(良好的自相关特性和处处为零的互相关特性)
  • 采用m序列作为地址码，必须选择互相关值很小 的m序列组，以避免用户之间的相互干扰
  • m序列升级：Gold码序列：优选对序列作模2加运算,生成的新的码序列即为Gold序列。Gold序列的序列数比m序列数多得多。

CDMA系统应用

三种序列分别是长码、短码和Walsh码

  • 基站信号：通过短PN码区分
  • 移动台：通过长PN码区分
  • 64个信道：正交的 Walsh码进行区分

基站地址--短码（中长m序列的截段码或Gold码）

用户地址码--长码（用长m序列的截段码）

信道地址码--64阶walsh函数

多载波技术

多载波传输是将高速串行的数据码流转换为N路并行的低速数据码流，然后调制到等频率间隔的N个相互正交的子载波上同时进行传输

OFDM(正交频分复用)

是一种特殊的多载波通信方案

各子载波上的频谱相互重叠

在频域内将信道划分成若干互相正交的子信道，每个子信道均拥有自己的载波，分别对其进行调制：

  • 正交信号可以通过在接收端采用相关技术来分开
  • 减少子信道之间的相互干扰

如果各个子信道的带宽被划分的足够窄，从而就可以消除符号间干扰，即每个子信道都可以看做无符号间干扰的理想信道。

缺点

存在较高的峰值平均功率比

  • 载波间干扰（ICI）（inter carrier interference）

保护间隔

通过引入保护间隔几乎可以完全消除符号间干扰

退避（backoff）过程

当媒体空闲时间≥IFS，立即传输

路由协议的分类

表驱动（Table Driven）路由

路由延时小，但是路由开销大

按需(On-demand)路由

路由延时大，但是路由开销小

DV算法

  • 基于分布式Bellman -Ford算法
  • 每个节点都维护一张路由表
  • 无法发现路由循环
  • 限制了网络的可扩展性
  • 水平分割 (split horizon)：当一个节点把路由更新发送给相邻节点时，它并不把从这个相邻节点处学到的路由再回送给该节点

DSDV协议

  • 基于DV算法
  • 确保无路由回路
  • 对拓扑变化能作出快速反应
  • 先验式（表驱动）路由
    • 路由信息必须周期性的更新（无休眠节点）

每个节点的路由表包括4部分：目的节点， 下一跳节点，度量值（距离值）和目的节点的序列号(偶数)。每次公告增加自己的目的地序列号（只使用偶数值）

节点不再可达：该节点的序列号加1（奇数序列号），并且设置metric为∞

DSDV路由选择

  • 选择具有更大目的地序列号的路由
  • 列号相等时，选择具有更好metric的路由

DSDV协议操作：减缓路由波动

在一个单独的表中记录每条路由的最近的和平均的Settling Time，

等待时间=2*(avg. Setting Time)

可缓解大型网络的路由波动问题， 从而避免不必要的公告，节约了带宽

按需（反应式）路由协议-动态源路由协议(DSR)

路由发现和路由维护都是按需进行的

  • 不需要周期性路由公告
  • 不需要感知链路状态
  • 不需要邻居检测

按需路由：

  • 节点间需要发送数据时才进行路由发送过程
  • 反应型路由，仅维护活跃的路由

源路由

  • 发送节点在分组中携带到达目的节点的路由信息，不需要中间节点维护路由信息。（类似数据报）
  • 节点缓存到目的节点的多条路由

DSR优化：路由缓存

设置缓存路由的有效期，过期即删除

什么是隐藏节点？什么是暴露节点？CSMA/CD协议为什么不能用于WLAN？

隐藏节点：在接收者的通信范围内而在发送者的通信范围外的终端。

（A向B发送，C听不到A的发送，C也发送，导致碰撞）

暴露节点：在发送者的通信范围之内而在接收者的通信范围之外的终端

（B向A发送，C听到B的发送，不能向D发送，造成不必要的延迟，因此C向D发送并不会影响B与A的通信）

CSMA/CD协议要求一个站点在发送本站数据的同时，还必须不断检测信道，但在无线局域网中实现这种功能就花费过大。 即使能够实现碰撞检测（CD）的功能，并且我们当发送数据时检测到信道是空闲的，由于隐藏节点和暴露节点的问题，在接收端仍然有可能发生碰撞。 因此WLAN使用CSMA，而非碰撞检测。

OFDM系统中存在哪两类干扰？4G是如何消除这两类干扰的？

OFDM（正交频分复用orthogonal frequency-division multiplexing）

符号间干扰ISI（inter symbol interference）和载波间干扰ICI（inter carrier interference）。

通过引入保护间隔几乎可以完全消除符号间干扰。

在其保护间隔内填入循环前缀信号（长度为保护间隔的长度）就不会在解调过程中产生子载波间干扰。

交织器作用：把可能存在的连续差错比特分散，以便于信道解码时纠错。纠正突发差错。

多普勒效应是什么和速度的关系

当移动台以恒定的速率沿某一方向移动时，由于传播路程差的原因，会出现合成频率在中心频率上下偏移的现象，称为多普勒效应。

速度越大，多普勒效应越明显。

4g和5g各种指标

峰值数据传输速率（Peak Data Transfer Rate）：这是网络能够提供的最高数据传输速率。在4G中，峰值速率通常为100 Mbps至1 Gbps，而在5G中，可以达到10 Gbps以上。

延迟（Latency）：延迟是指数据从发送到接收所需的时间。在4G中，延迟通常为10毫秒至50毫秒，而在5G中，延迟可以降低到1毫秒以下。

频谱效率（Spectral Efficiency）：频谱效率是指在特定频段内传输的数据量。5G比4G更高效，可以实现更高的频谱效率，从而在相同的频谱资源下提供更多的数据传输。

连接密度（Connection Density）：连接密度是指网络可以同时支持的设备数量。5G相对于4G具有更高的连接密度，可以同时连接更多的设备。

覆盖范围（Coverage Range）：覆盖范围是指网络信号可以传播的距离。尽管5G的覆盖范围较4G更小，但5G可以通过使用更多的小型基站来提供更广泛的覆盖。

流量密度

4g和5g带宽、速率、间隔长度

带宽：4G网络的带宽通常为20 MHz，而5G网络的带宽可以达到100 MHz或更高。这意味着5G网络具有更大的带宽，可以支持更多的数据传输。

速率：4G网络的理论最高下载速率为100 Mbps（兆比特每秒），而5G网络的理论最高下载速率可以达到10 Gbps（千兆比特每秒）。 这意味着5G网络具有更快的传输速度，可以更快地下载和上传数据。

间隔长度：4G网络的子载波间隔长度为15 kHz，而5G网络的子载波间隔长度为15 kHz或更小。这意味着5G网络可以在更小的频谱间隔内传输更多的数据，从而提高了频谱利用率。

一、Commando War

There is a war and it doesn’t look very promising for your country. Now it’s time to act. You have a commando squad at your disposal and planning an ambush on an important enemy camp located nearby. You have N soldiers in your squad. In your master-plan, every single soldier has a unique responsibility and you don’t want any of your soldier to know the plan for other soldiers so that everyone can focus on his task only. In order to enforce this, you brief every individual soldier about his tasks separately and just before sending him to the battlefield. You know that every single soldier needs a certain amount of time to execute his job. You also know very clearly how much time you need to brief every single soldier. Being anxious to finish the total operation as soon as possible, you need to find an order of briefing your soldiers that will minimize the time necessary for all the soldiers to complete their tasks. You may assume that, no soldier has a plan that depends on the tasks of his fellows. In other words, once a soldier begins a task, he can finish it without the necessity of pausing in between.

Input: There will be multiple test cases in the input file. Every test case starts with an integer N(1 <= N <= 1000), denoting the number of soldiers. Each of the following N lines describe a soldier with two integers B(1 <= B <= 10000)&J(1 <= J <= 10000). B seconds are needed to brief the soldier while completing his job needs J seconds. The end of input will be denoted by a case with N = 0 . This case should not be processed.

Output: For each test case, print a line in the format, “Case X: Y ”, where X is the case number &Y is the total number of seconds counted from the start of your first briefing till the completion of all jobs.

1.自然语言描述和伪代码

伪代码

自然语言描述

  1. 输入处理： 对于每个测试用例，首先读取 N，表示士兵的数量。如果 N 为 0，则结束输入。
  2. 士兵信息读取： 读取接下来的 N 行，每行包含两个整数 B 和 J，分别表示士兵的说明时间和完成任务时间。
  3. 排序： 对士兵数组按照说明时间升序和完成任务时间降序进行排序，以确保最优的任务执行顺序。
  4. 计算总时间： 初始化总时间（total_time）和当前时间（current_time）为 0。遍历排序后的士兵数组，累加每个士兵的说明时间到当前时间，并更新总时间为当前时间加上该士兵完成任务的时间的最大值。
  5. 输出结果： 输出当前测试用例的结果，格式为“Case X: Y”，其中 X 是测试用例编号，Y 是从第一个说明开始到完成所有任务所需的总时间。

2.贪心思路

算法是利用了排序解决了“贪心“的思路。

因为在所有士兵中，说明时间越短一定越有利于节省时间，二在说明时间相同的情况下，完成任务时间最长的士兵可以更早的开始任务，以便于更早 的结束任务，可以在总体上花费最短的时间

3.算法的正确性证明

  1. 初始化步骤： 对于排序后的士兵数组，初始的总时间是0。
  2. 归纳假设： 设当前已经选择了前k个士兵，总时间为T。假设这个选择是最优的，即不存在其他选择使得总时间更短。
  3. 归纳证明： 考虑加入第k+1个士兵，这时选择的是说明时间最短的、完成任务时间最长的士兵。这样选择的合理性在于，希望尽早完成说明，以便后续的士兵可以更早开始执行任务，从而最终减少总时间。
    • 如果选择了其他士兵而不是第k+1个士兵，假设选择了第i个士兵，那么说明时间就会更长。因为选择的是完成任务时间最长的，所以其他士兵的完成任务时间一定更短，而完成任务时间短的士兵说明时间更短，所以总时间会更长。
  4. 归纳步骤： 继续这个过程，每次选择说明时间最短的、完成任务时间最长的士兵，直到选择完所有士兵。这样的选择策略保证了总时间最短。

由此，证明了贪心算法的正确性。

4.算法的复杂度

  • 排序复杂度： 对于N名士兵，进行一次排序的时间复杂度为O(NlogN)。
  • 总时间计算复杂度： 在排序后的数组中遍历每个士兵，计算总时间。这一步的时间复杂度是O(N)。

二、DNA Consensus String

The Hamming distance is the number of different characters at each position from two strings of equal length.

For example, assume we are given the two strings “AGCAT” and “GGAAT.” The Hamming distance of these two strings is 2 because the 1st and the 3rd characters of the two strings are different.

Using the Hamming distance, we can define a representative string for a set of multiple strings of equal length.

Given a set of strings S = {s1, ... , sm} of length n, the consensus error between a string y of length n and the set S is the sum of the Hamming distances between y and each si in S. If the consensus error between y and S is the minimum among all possible strings y of length n, y is called a consensus string of S.

For example, given the three strings “AGCAT”、“AGACT” and “GGAAT” the consensus string of the given strings is “AGAAT” because the sum of the Hamming distances between “AGAAT” and the three strings is 3 which is minimal. (In this case, the consensus string is unique, but in general, there can be more than one consensus string.)

We use the consensus string as a representative of the DNA sequence. For the example of Figure 1 above, a consensus string of gene X is “GCAAATGGCTGTGCA” and the consensus error is 7.

Input: Your program is to read from standard input. The input consists of T test cases. The number of test cases T is given in the first line of the input. Each test case starts with a line containing two integers m and n which are separated by a single space. The integer m(4 ≤ m ≤ 50) represents the number of DNA sequences and n(4 ≤ n ≤ 1000) represents the length of the DNA sequences, respectively. In each of the next m lines, each DNA sequence is given.

Output: Your program is to write to standard output. Print the consensus string in the first line of each case and the consensus error in the second line of each case. If there exists more than one consensus string, print the lexicographically smallest consensus string.

1.自然语言描述和伪代码

伪代码

自然语言描述

  1. 输入： 读取测试用例数 T。
  2. 循环处理每个测试用例：
    • 读取整数 m 和 n，表示 DNA 序列的数量和长度。
    • 初始化一个空列表 sequences 用于存储 DNA 序列。
  3. 处理每个 DNA 序列的输入：
    • 通过循环读取每个 DNA 序列的字符，将其追加到列表 sequences 中。
  4. 寻找共识序列：
    • 对于每个位置 i（从 1 到 n），初始化一个字典 counts 用于统计 A、C、G、T 四个字符的出现次数。
    • 对于每个 DNA 序列，统计当前位置字符的出现次数，并选择出现次数最多的字符作为目标序列的当前位置字符。
  5. 计算 Hamming 距离：
    • 遍历每个 DNA 序列，计算其与目标序列的 Hamming 距离，并累加得到总的 Hamming 距离。
  6. 输出结果：
    • 输出目标序列。
    • 输出总的 Hamming 距离。

2.贪心思路

贪心选择策略：

  • 在每个位置上选择出现次数最多的字符，目的是试图在每个位置上使目标序列与输入序列的差异尽可能小，从而最小化总的 Hamming 距离。

如何想到使用贪心

  • 考虑从每个位置的字符出现次数的角度出发，贪心地选择每个位置的字符，试图最小化总的 Hamming 距离。这是因为在每个位置上选择出现次数最多的字符，相当于局部最优的选择，希望通过这种局部最优选择达到全局最优。
  • 具体地，通过统计每个位置上字符的出现次数，选择出现次数最多的字符，将其作为目标序列的字符。这样的选择策略使得目标序列在每个位置上尽量与输入序列相似，从而最小化 Hamming 距离。
  • 每个位置的字符选择可以看作是一个局部最优解，通过选择每个位置上的字符，希望达到总体上的最优解，即最小化总的 Hamming 距离。

3.算法的正确性证明

归纳基础：

首先证明当序列的长度 n 为 1 时算法的正确性。

  1. 初始情况： 当 n=1 时，只有一个字符，无需比较，算法直接输出该字符，这是显然正确的。

归纳步骤：

假设当序列的长度为 n=k 时，算法对任意长度为 k 的 DNA 序列集合都能正确地找到共识序列。

现在考虑长度为 n=k+1 的情况。将算法的步骤分解为以下几个部分：

  1. 寻找共识序列： 对于每个位置 i（从 1 到 k+1），统计出现次数最多的字符作为目标序列的当前位置字符。
    • 根据归纳假设，算法在长度为 k 的情况下是正确的。
    • 添加第 k+1 个位置的字符时，根据算法，选择出现次数最多的字符作为目标序列的字符。
  2. 归纳步骤： 根据归纳假设，算法对长度为 k 的情况是正确的，现在要证明它对长度为 k+1 的情况也是正确的。
    • 在选择第 k+1 个位置字符时，算法选择的是出现次数最多的字符，这保证了在每个位置上都选择了局部最优的字符。
    • 通过局部最优的选择，期望在整体上得到总体最优解，即最小化总的 Hamming 距离。

因此，通过数学归纳法，证明了算法对于任意长度的 DNA 序列集合都能正确地找到共识序列

4.算法的复杂度

时间复杂度分析：

  1. 寻找共识序列 (FindConsensus): 对于每个位置 i，都需要统计 m 个序列在该位置上的字符出现次数，然后选择出现次数最多的字符。这一过程的时间复杂度为 \[ O(m \cdot n) \] 其中 n 是序列的长度。

  2. 计算错误 (CalculateError): 需要遍历 m 个序列，每个序列长度为 n，并计算 Hamming 距离。计算 Hamming 距离的时间复杂度为 O(n)。因此，计算错误的总时间复杂度为 \[ O(m \cdot n)。 \]

因此，整体上，算法的时间复杂度为 \[ O(m \cdot n)。 \] 空间复杂度分析：

  1. ****寻找共识序列 (FindConsensus):需要一个长度为 n 的字符串 (consensus) 用于存储共识序列，以及一个长度为 4 的数组 (counts) 用于统计字符出现次数。因此，空间复杂度为 O(n)。
  2. ****计算错误 (CalculateError): 额外的空间复杂度主要来自常数大小的变量和函数调用栈，可以忽略。

因此，整体上，算法的空间复杂度为 O(n)。

综上所述，算法的时间复杂度为 \[ O(m \cdot n) \] 空间复杂度为 \[ O(n) \]

三、Opponents

Arya has n opponents in the school. Each day he will fight with all opponents who are present this day. His opponents have some fighting plan that guarantees they will win, but implementing this plan requires presence of them all. That means if one day at least one of Arya’s opponents is absent at the school, then Arya will beat all present opponents. Otherwise, if all opponents are present, then they will beat Arya. For each opponent Arya knows his schedule — whether or not he is going to present on each particular day. Tell him the maximum number of consecutive days that he will beat all present opponents.

Note, that if some day there are no opponents present, Arya still considers he beats all the present opponents.

Input: The first line of the input contains two integers n and d ( 1 <= n, d <= 100 ) — the number of opponents and the number of days, respectively. The i -th of the following d lines contains a string of length n consisting of characters ’0’ and ’1’. The j -th character of this string is ’0’ if the j -th opponent is going to be absent on the i -th day.

Output: Print the only integer — the maximum number of consecutive days that Arya will beat all present opponents.

1.自然语言描述和伪代码

伪代码

自然语言描述

需要找到 Arya 连续击败所有在场对手的最长天数。对于每一天，如果至少有一个对手不在场，Arya 就能赢；否则，如果所有对手都在场，Arya 就输了。目标是找到 Arya 连续赢的最长天数。

可以使用滑动窗口的方法。维护一个窗口，窗口的大小就是 Arya 连续击败所有在场对手的天数。在窗口中检查每一天的情况，如果有一天至少有一个对手不在场，就移动窗口的右边缘；否则，移动窗口的左边缘。在这个过程中，保持记录最大窗口的大小。

2.贪心思路

Arya 想要最大化连续击败对手的天数。贪心的思路是在每一步都做出当前最优的选择，即在每一天都选择击败对手，只要有对手在场。

思考贪心的过程：

  1. 问题特性： 首先观察问题的特性，Arya 在一天内要么赢，要么输。他要连续击败对手，就不能有输的情况。
  2. 最优子结构： Arya 连续击败对手的最长天数应该是在每一步都做出当前最优的选择，即每一天都击败对手。
  3. 贪心策略： 可以贪心地选择每一天都击败对手。如果有一天没有对手在场，那么这一天是连续击败对手的开始，就从下一天重新计数。

贪心的实现（滑动窗口）：

  1. 使用一个窗口，窗口的大小表示 Arya 连续击败对手的天数。
  2. 遍历每一天，如果有一天至少有一个对手不在场，窗口右移；否则，窗口左移。
  3. 在每一步都更新最大连续击败天数。

为什么使用滑动窗口：

滑动窗口适用于连续子序列的问题。在这个问题中，需要找到连续的击败对手的天数，因此滑动窗口是一个自然的选择。窗口的左右边缘表示 Arya 连续击败对手的起始和结束位置，通过滑动窗口来寻找最优的连续天数。

3.算法的正确性证明

基础步骤： 对于 d=1，即一天的情况，算法首先检查对手是否存在。如果对手存在，Arya将被击败；否则，他将战胜所有对手。这是正确的，因为算法会返回0，即Arya在这一天战胜所有对手的最大连续天数为0。

归纳步骤： 假设对于规模为 d=k 的问题，算法给出的最大连续击败天数是正确的。现在考虑规模为d=k+1 的问题。

如果在第k+1 天对手的存在数量小于n，那么 Arya 将战胜所有对手，并且最大连续击败天数将在前一天的基础上加1。这符合归纳假设。

如果在第k+1 天对手的存在数量等于n，那么 Arya 将被击败，最大连续击败天数将重置为0。这也是正确的，因为在这一天 Arya 未能击败所有对手。

因此，通过数学归纳法，可以证明算法对于所有符合条件的输入是正确的。

4.算法的复杂度

时间复杂度

  1. CountOpponentsPresent 函数的复杂度：

    • 在每一天的循环中，CountOpponentsPresent 函数对对手的状态进行遍历。
    • 时间复杂度：O(n)

  2. MaxConsecutiveDays 函数的复杂度：

    • MaxConsecutiveDays 函数包含一个从 1到 d 的天数循环，每次迭代中调用了 CountOpponentsPresent 函数。

    • 因此，总的时间复杂度为： \[ O(d⋅n) \]

空间复杂度：除了输入参数外，算法使用的额外空间非常有限，只有几个整数变量，因此整体空间复杂度是常数级的。 \[ O(1) \]

四、Minimum Varied Number

Find the minimum number with the given sum of digits s such that all digits in it are distinct (i.e. all digits are unique). For example, if s = 20 , then the answer is 389 . This is the minimum number in which all digits are different and the sum of the digits is 20 ( 3 + 8 + 9 = 20 ). For the given s print the required number.

Input: The first line contains an integer t ( 1 ≤ t ≤ 45 ) — the number of test cases. Each test case is specified by a line that contains the only integer s ( 1 ≤ s ≤ 45 ).

Output: Print t integers — the answers to the given test cases.

1.自然语言描述和伪代码

伪代码

自然语言描述

  • 目标： 在给定数字和 s 的情况下，目标是构造一个具有唯一数字的最小数字。
  • 选择最大的数字： 为了使数字最小，应该尽可能选择高位上的最大数字。
  • 递减选择： 可以从 9 开始，递减到 1，依次尝试将最大的数字添加到数字中，同时保持数字和不超过 s。
  • 保持唯一性： 还需要确保添加的数字在已经选择的数字中是唯一的，以满足“所有数字都是唯一的”这个条件。
  • 算法步骤：
    • 从 9 开始递减选择数字。
    • 对于每个数字，检查将其添加到当前数字中是否会导致数字和超过 s。
    • 如果不超过，将该数字添加到数字中。
    • 重复此过程，直到达到给定数字和 s。

2.贪心思路

贪心选择的原因：

  • 通过选择高位上的最大数字，可以确保数字是最小的。这是因为高位上的数字对数字的大小有更大的影响。
  • 通过递减选择，保持了数字和的最小性，因为首先考虑了最大的数字，然后是次大的数字，以此类推。
  • 确保唯一性是为了满足题目中的条件，即所有数字都是唯一的。

如何想到使用贪心：

  • 题目中要求构造最小数字，而最小数字通常与高位上的大数字有关。
  • 需要在数字和不超过给定和的情况下进行选择，这暗示了递减选择的过程。
  • 考虑到唯一性的要求，可以通过递减选择高位上的数字，以确保数字的唯一性。

综上所述，通过观察题目的特征和要求，可以想到使用贪心策略来构造最小的、具有唯一数字的数字。

3.算法的正确性证明

  1. 基础步骤：

    1. 初始情况： 对于 s = 1，最小数字是 1，唯一且符合条件。
    2. 初始步骤： 假设对于 s = k（其中 k 为某个正整数），贪心算法构造的数字是最小的、具有唯一数字的数字。

  2. 归纳步骤：

    1. 递增情况： 考虑 s = k + 1 的情况。 s = k 时的最小数字，即已构造的数字。
    2. 添加下一位： 贪心算法会尝试添加下一位，即从 9 到 1 中选择一个数字，以保持数字和不超过 s。
    3. 保持唯一性： 由于在构造 s = k 时已经考虑了唯一性，所以在添加下一位时，贪心算法会选择保持已经选择数字的唯一性。
    4. 保持最小性： 在添加下一位时，贪心算法会选择尽可能大的数字，以保持数字的最小性。
    5. 综合证明： 综合考虑递增情况的所有步骤，得出结论：对于 s = k + 1，贪心算法构造的数字仍然是最小的、具有唯一数字的数字。

    结论： 根据数学归纳法，证明了对于任意正整数 s，贪心算法构造的数字是最小的、具有唯一数字的数字。

4.算法的复杂度

时间复杂度： O(t * 9) ，在最坏的情况下，从左到右迭代每个数字，对于每个数字，都有一个最多9次迭代的循环。

空间复杂度： O(1) ，使用了常量空间

五、Minimum Varied Number

Joey Takes Money Joey is low on money. His friend Chandler wants to lend Joey some money, but can’t give him directly, as Joey is too proud of himself to accept it. So, in order to trick him, Chandler asks Joey to play a game. In this game, Chandler gives Joey an array a1, a2, . . . , an ( n ≥ 2 ) of positive integers ( ai ≥ 1 ). Joey can perform the following operation on the array any number of times:

Take two indices i and j ( 1 ≤ i < j ≤ n) . 2. Choose two integers x and y ( x, y ≥ 1 ) such that x · y = ai · aj . 3. Replace ai by x and aj by y . In the end, Joey will get the money equal to the sum of elements of the final array.

Find the maximum amount of money ans Joey can get but print 2022 · ans . Why multiplied by 2022 ? Because we are never gonna see it again! It is guaranteed that the product of all the elements of the array a doesn’t exceed 1012.

Input: Each test contains multiple test cases. The first line contains the number of test cases t ( 1 ≤ t ≤ 4000 ). Description of the test cases follows. The first line of each test case contains a single integer n ( 2 ≤ n ≤ 50 ) — the length of the array a . The second line contains n integers a1, a2, . . . , an ( 1 ≤ ai ≤ 106 ) — the array itself. It’s guaranteed that the product of all ai doesn’t exceed 1012 (i. e. a1 · a2 · . . . · an ≤ 1012 ).

Output: For each test case, print the maximum money Joey can get multiplied by 2022 .

1.自然语言描述和伪代码

伪代码

自然语言描述

  1. 对数组进行排序，从小到大。
  2. 初始化变量 maxMoney 为 1，表示 Joey 能得到的最大金额。
  3. 从数组的最小值开始遍历，设当前元素为 currentElement。
  4. 对于 currentElement的每个因子对 (x, y)，其中 x * y = currentElement：
    • 计算 Joey 得到的金额 currentMoney = x + y。
    • 如果 currentMoney 大于 maxMoney，更新 maxMoney 为 currentMoney。
  5. 继续遍历数组的下一个元素，重复步骤 3-4。
  6. 最终，maxMoney 的值即为 Joey 能得到的最大金额。

2.贪心思路

问题要求 Joey 最大化他得到的钱，而 Chandler 给了 Joey 一个数组，Joey可以通过选择数组中的两个元素，将它们替换为它们的因子，从而最终得到数组中所有元素的和。目标是找到 Joey 能够获得的最大钱数。

贪心思路步骤：

  1. 对数组进行排序： 将数组按升序排序，以便 Joey 从小的元素开始选择。
  2. 遍历数组元素： 对于数组中的每个元素，考虑替换它为两个因子的组合，使得替换后的和最大。
  3. 选择最大的因子组合： 对于当前元素，选择它的两个因子组合，使得组合的和最大。可以通过计算元素的所有因子对，并选择使和最大的组合。
  4. 更新最大值： 在每一步中，更新当前的最大值。
  5. 返回结果： 返回最终得到的最大值。

为什么使用贪心？

在这个问题中，每一步选择是选择一个元素的两个因子，以使得替换后的和最大。通过贪心地选择使得每一步都达到局部最优解，可以期望得到全局最优解。

贪心的思路是尽可能地选择大的因子，以获得更大的和。在排序数组后，从小到大考虑每个元素，选择其两个因子中最大的组合，这样就可以保证每一步都是局部最优的。

3.算法的正确性证明

假设 Joey 在每一步中都选择了最优的替换方案，即选择了两个因子中的最大组合，以最大化替换后的和。

基础步骤： 对于数组长度为 2 的情况，算法直接返回数组中两个元素的和，这就是最大值。这个基础步骤是正确的。

归纳假设： 假设算法对于数组长度为 k 的情况是正确的，即在数组中选择了最优的替换方案，以最大化替换后的和。

归纳步骤： 考虑数组长度为 k+1 的情况。在排序后的数组中，选择最小的两个元素 a 和 b 进行替换。根据归纳假设，对于数组中除了 a 和 b 的部分，算法选择了最优的替换方案。

现在要证明，在包含 a 和 b 的情况下，选择 a 和 b 的最大因子组合也是最优的。设 a = p1 * q1，b = p2 * q2，其中 p1, q1, p2, q2 是 a 和 b 的因子。

由于数组的元素都是正整数，p1, q1, p2, q2 都是大于等于 1 的整数。要证明选择 p1, q2 或者 p2, q1 中的最大组合，是为了最大化 a+b 的和。

考虑两种情况：

  1. 如果 p1 >= p2，q1 >= q2，即 a 的两个因子都大于等于 b 的两个因子。那么选择 p1, q2 作为替换组合，a + b 的和是 (p1 * q1) + (p2 * q2) >= (p1 * q2) + (p2 * q2) = (p1 + p2) * q2。由于 p1 + p2 >= 2，所以 (p1 + p2) * q2 >= 2 * q2 >= q1 + q2。因此，选择 p1, q2 是为了最大化和。
  2. 如果 p1 >= p2，q1 < q2，即 a 的两个因子中，第一个因子大于等于 b 的两个因子，第二个因子小于 b 的两个因子。那么选择 p1, q2 作为替换组合，a + b 的和是 (p1 * q1) + (p2 * q2) >= (p1 * q2) + (p2 * q2) = (p1 + p2) * q2。由于 p1 + p2 >= 2，所以 (p1 + p2) * q2 >= 2 * q2 > q1 + q2。因此，选择 p1, q2 是为了最大化和。

综上所述，无论是哪种情况，选择 a 和 b 的最大因子组合都是为了最大化和。因此，基于归纳假设，证明了算法对于数组长度为 k+1 的情况也是正确的。

由归纳法可知，算法对于任意长度的数组都是正确的。

4.算法的复杂度

时间复杂度：

  1. 排序数组的时间复杂度为 O(n log n)，其中 n 为数组的长度。
  2. 遍历数组的时间复杂度为 O(n)。

因此，整体时间复杂度为 O(n log n)。

空间复杂度：

空间复杂度取决于排序算法的实现。 如果使用的是不占用额外空间的排序算法（比如堆排序），则空间复杂度为 O(1)。 如果使用的是需要额外空间的排序算法（比如归并排序），则空间复杂度为 O(n)。
